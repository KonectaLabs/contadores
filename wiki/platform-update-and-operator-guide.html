Lo que falta para pasar de staged/dry-run a ejecucion externa real.

Credenciales faltantes
Meta Ads

Faltan env vars en el server

Estas variables aparecen vacias en produccion. Con esto cargado, la plataforma puede leer inventario, buscar leads por API, subir creatividades y ejecutar planes aprobados.

META_MARKETING_API_VERSION META_MARKETING_ACCESS_TOKEN META_ACCESS_TOKEN META_AD_ACCOUNT_ID META_MARKETING_LIVE_WRITES_ENABLED
Accion siguiente

Conseguir token y ad account

Usar 1Password/API si existe. Si no tenemos acceso, pedirle a Alan el token, el ad account y confirmacion de activar live writes.

Google Calendar

Faltan env vars de calendario

El scheduler esta preparado para dry-run y para guardar blockers, pero crear eventos reales requiere estas variables.

PLATFORM_MEETING_CALENDAR_ID PLATFORM_MEETING_INTERNAL_ATTENDEES GOOGLE_CALENDAR_SERVICE_ACCOUNT_JSON GOOGLE_CALENDAR_DELEGATED_USER

Definir calendario operativo

Decidir en que calendar se crean reuniones, quienes van como asistentes internos y con que usuario delegado escribe Google.

Guia de usuario para Mati

Flow operativo para recibir leads, revisar clientes y controlar que todo corra.

Empleado operador

Abrir la plataforma

Entrar a https://crm.fgoiriz.com. En local de desarrollo, usar http://localhost:5173/. Arriba estan las secciones principales: CRM, Workstation, Delivery, Ops y Runner.

Empezar por Ops

Tocar Ops y despues Refresh. Mirar primero Next best action, Action queue y Meta readiness. Si hay un blocker, no asumir: leer el texto y escalarlo.

Responder leads en CRM

Ir a CRM, elegir el funnel arriba (Contadores, Abogados, etc.) y filtrar por Needs answer. Abrir cada lead, leer Messages y usar Send para responder.

Decidir el estado del lead

Usar Answered si ya se respondio, Manual para dejarlo en manejo humano, Converted si ya avanzo a reunion o Workstation, Close si no va mas, y Reopen si hay que revivir uno cerrado.

Usar Codex con cuidado

El switch Codex activa/desactiva ayuda AI para ese lead. Si el lead pregunta algo delicado o falta informacion, pausarlo con Manual y pedir contexto a Facu.

Convertir un cliente

Si el lead pago o ya es cliente, tocar Convert. Si solo compro una pagina, tocar Solo page. Despues seguirlo desde Workstation.

Trabajar clientes en Workstation

Ir a Workstation, elegir el cliente, revisar Meeting notes, subir archivos en Client files, copiar contexto con Copy all y bajar entregables con el link de ZIP cuando este disponible.

Revisar Delivery de leads a clientes

Ir a Delivery, elegir el contacto del cliente y mirar Leads from sheet. Los filtros All sheets y cada sheet/campana separan las fuentes. No hay boton manual de sync: al seleccionar contacto refresca solo.

Resolver envios fallidos

En Delivery mirar Sent chat, Delivery y las columnas blocked/failed. Usar Retry solo si el error es retryable. Usar Copy o Copy all para pasar datos a Facu.

Mirar Runner si algo se queda parado

Runner muestra si hay procesos agenticos corriendo, idle o fallando. Si esta en failed/error, sacar contexto de Ops antes de tocar configuracion.

Meta Ads flow

Como deberia funcionar cuando el empleado pide ads y cuando se habiliten credenciales.

Publicacion real bloqueada hasta API keys
1. Brief

Transcript o notas

El agente guarda transcript/notas del cliente, extrae perfil, oferta, objeciones, geografia, segmentos y angulos de ads.

2. Campana

Stage ad campaign

Se guarda objetivo, presupuesto, segmentos, angulos, creative_benchmark y creative_testing. Default: Eliana v3 y 3 conceptos x 10 variantes.

3. Creatividades

Generar batch, no elegir a ojo

Por cada concepto se generan 10 variantes. Cada imagen aceptada se guarda como CreativeAsset. La idea no es elegir una sola manualmente: Meta deberia optimizar.

4. Inventario

Sync Meta inventory

Con credenciales, la plataforma lee ad account, Page, formularios, pixels, WhatsApp numbers y campanas existentes. Sin credenciales, Ops muestra missing_credentials.

5. Plan

Stage + preflight

Se arma el plan Campaign -> Ad Set -> Creative -> Ad en estado PAUSED. Preflight valida campos, ruteo, presupuesto, approval y creatividades Meta-ready.

6. Approval

Facu aprueba presupuesto

Antes de live write, Facu o un operador autorizado aprueba caps, nota, idempotency key y que todo arranque pausado.

7. Execute

Crear objetos reales en Meta

Solo con META_MARKETING_LIVE_WRITES_ENABLED=true, token y approval. El executor guarda IDs devueltos por Meta para no duplicar en retries.

8. Leads

Volver al CRM o Delivery

Click-to-WhatsApp vuelve al funnel por source/ad ID. Instant forms entran a Client Lead Delivery por client_lead_source_id.

9. Updates

Reporte cada 24 horas

Ops y client updates deben mostrar leads, delivery, blockers, Meta status y proximo paso para mandar un update claro al cliente.

Regla creativa para Mati No pedir "haceme un ad lindo". Pedir el batch completo: 3 conceptos concretos del problema del cliente, 10 variantes por concepto, estilo Eliana v3, problema visible primero, persona visible, texto grande y WhatsApp/legal secundario.

Rutina diaria recomendada

Checklist operativo simple para que Mati no se pierda.

15 minutos al empezar
Manana

Primer chequeo

  1. Abrir Ops y tocar Refresh.
  2. Revisar Blockers y Questions.
  3. Entrar a CRM y filtrar Needs answer.
  4. Responder o pausar leads que no pueda resolver.
  5. Entrar a Delivery y mirar failed/blocked.
Durante el dia

Operacion normal

  1. Cuando entra un lead importante, revisar mensajes completos antes de responder.
  2. Si hay intencion real o pago, marcar Converted solo cuando tenga datos reales.
  3. Si pago, usar Convert y mover a Workstation.
  4. Si un cliente pregunta por leads, revisar Delivery y Sent chat.
  5. Si Meta esta bloqueado por credenciales, no inventar IDs: escalar.
Antes de cerrar

Control final

  1. Volver a Ops y confirmar blockers nuevos.
  2. Revisar Client updates para updates de 24 horas.
  3. Dejar preguntas abiertas con contexto claro para Facu.
  4. No dejar leads buenos en Needs answer si ya se respondieron.
No tocar sin permiso

Guardrails

  • No borrar leads, funnels o delivery sources sin confirmar.
  • No cambiar Runtime, Funnel o Config si no sabe por que.
  • No aprobar Meta live writes sin presupuesto y OK explicito.
  • No elegir una sola creatividad ganadora antes de datos de Meta.
