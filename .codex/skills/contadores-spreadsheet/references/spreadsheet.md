## What The Spreadsheet Represents

This sheet contains inbound leads generated by Meta lead forms for the `contadores` offer.

Right now it already captures:

- lead identity;
- lead creation timestamp;
- ad, ad set, campaign, and form metadata;
- channel information such as `fb` or `ig`;
- contact data like email, name, and phone number;
- a coarse operational flag for whether the lead was already contacted.

The plan is to extend the same sheet so it also stores WhatsApp automation state.

## How To Connect From Python

### Option 1: Public read

If the sheet is public, the simplest path is:

```bash
cd /Users/fgoiriz/private/repos/contadores
uv run python read_google_sheet.py --as-records
```

The script uses:

- `GOOGLE_SHEET_URL`
- `GOOGLE_SHEET_GID`

from `.env`, so `--sheet` is optional.

Internally it tries:

```text
https://docs.google.com/spreadsheets/d/<spreadsheet_id>/export?format=csv&gid=<gid>
```

and parses the CSV output.

### Option 2: Service account

Use this for private access or any write workflow.

Required steps:

1. Enable Google Sheets API in Google Cloud.
2. Create a service account.
3. Download the JSON key.
4. Share the sheet with the service account as `Editor`.
5. Set `GOOGLE_SERVICE_ACCOUNT_FILE` to the JSON path.

Example:

```bash
GOOGLE_SERVICE_ACCOUNT_FILE=/path/to/service-account.json \
uv run python read_google_sheet.py --as-records
```

For this project, the sheet is useful because it is:

- visible to non-technical operators;

## Meaning Of Proposed Automation Columns

- `wa_status`: High-level state such as `pending`, `sent`, `waiting_reply`, `done`, `error`, `handoff`.
- `wa_sequence`: Which sequence the lead is following.
- `wa_step`: Current step in that sequence.
- `wa_last_outbound_at`: Timestamp of the last outbound WhatsApp message.
- `wa_last_inbound_at`: Timestamp of the last inbound reply.
- `wa_next_action_at`: When the poller should next act on this lead.
- `wa_message_id`: Provider message id of the latest outbound message.
- `wa_error`: Last send or sync error.
- `wa_lock_token`: Short-lived claim token used by the worker.
- `wa_lock_until`: Lock expiration timestamp.
- `wa_human_handoff`: Boolean flag to stop automation and let a human continue.
- `wa_notes`: Freeform operator notes.

## Recommended Processing Flow

Every 5 minutes:

1. Read the rows.
2. Find leads eligible for action.
3. Skip rows with active locks.
4. Claim one row by writing `wa_lock_token` and `wa_lock_until`.
5. Send the WhatsApp message.
6. Only if send succeeds, update `is_contactado`, `wa_status`, `wa_step`, and `wa_next_action_at`.
7. If the lead replies, set `wa_human_handoff = TRUE` unless a later version supports automated branching.

## Idempotency Rules

- Never send based only on `is_contactado`.
- Use both sequence state and a lock field.
- Only move the step forward after a successful provider response.
- Keep provider ids in the sheet so retries can be reconciled.

## Data Hygiene Notes

- The sheet contains personal data.
- Do not keep it public in production.
- Normalize phone numbers before sending WhatsApp messages.
- Strip prefixes like `p:` before using the number downstream.
- Prefer ISO-like timestamp strings so operators can inspect them easily.
